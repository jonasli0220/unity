# 抽卡回归测试工具

用于 Dragonheir Windows Development Build 的“日芒召唤”连续十连 UI 回归测试。

## 能做什么

- 自动按顺序执行：召唤 10 次 → 跳过动画 → 等待结果页 → 下一次十连。
- 以游戏客户区比例定位按钮，窗口大小变化后无需重新录坐标。
- 点击落点只在按钮中央安全区内变化，用于覆盖按钮热区并减少边缘误触。
- 最后一次十连结束后停在结果页，不会多点下一次。
- 游戏窗口离开前台、窗口异常或按钮识别超时后自动暂停。
- 可选后台实验模式：不抢鼠标焦点，可在使用其他应用时继续执行。

本工具不读取或修改游戏内存，不注入游戏进程，不隐藏自身，也不提供反检测或安全绕过功能。

## 使用方法

1. 打开 Development Build，进入“英雄召唤 → 日芒召唤”首页，确认右下角能看到“召唤 10 次”。
2. 双击 `Start-AutoGachaQA.cmd`。
3. 填写“十连次数”。例如填 `10`，表示总共执行 100 抽。
4. 勾选 Development Build 测试环境确认框。
5. 点击“开始”，之后暂时不要操作鼠标。
6. 工具完成后会停在最后一轮结果页并播放提示音。

### 后台运行

勾选“后台运行（实验）”后，工具会把标准鼠标窗口消息直接发给指定的 Dragonheir 窗口，不移动系统鼠标，也不抢占前台焦点。

- 游戏窗口可以被其他窗口遮挡，但不能最小化。
- 工具优先尝试后台捕获画面并识别按钮。
- 部分 Unity/DirectX 客户端不能提供后台画面；此时工具会在日志中提示，并自动改用较保守的固定等待。
- 当前客户端是否接受后台窗口消息需要实测，第一次务必只设为 `1` 次十连，确认首页、跳过和结果页流程均正确后再增加次数。
- 后台模式只作用于进程名 `Dragonheir` 且标题严格匹配“龙息: 神寂”的窗口，不会向当前正在使用的其他应用点击。

## 随时控制

- `F8`：暂停 / 继续。
- `F9`：立即停止。
- 前台模式下主动切换到其他窗口：工具自动暂停，不会在其他应用中误点。
- 后台模式下可以正常切换应用；若将游戏最小化，工具会自动暂停。

继续前请先把游戏恢复到当前流程应处的界面；后台模式还需要先恢复已最小化的游戏窗口，再按 `F8`。

## 识别与坐标基准

工具按窗口客户区百分比定位，并用红色按钮区域做轻量状态识别。当前流程基于以下三个界面：

- 日芒召唤首页：右下角“召唤 10 次”。
- 召唤动画：右上角“跳过”。
- 抽卡结果页：底部“召唤 10 次”。

如果 UI 改版、按钮换色或位置移动，工具会在 20～25 秒后暂停并给出原因。此时不要反复继续，应联系维护者更新 `AutoGachaQA.ps1` 顶部的 `$Regions` 和识别阈值。

## 首次验证建议

UI 改版后先把“十连次数”设为 `1` 做一次验证，确认：

- 首页按钮点击正确；
- 动画页能正确跳过；
- 结果页被识别；
- 工具停在结果页，没有继续消耗下一次十连。
