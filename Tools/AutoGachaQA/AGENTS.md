# 抽卡回归测试工具规则

## 用途边界

- 本目录只存放 Dragonheir Development Build 的 UI 回归测试工具。
- 工具只能通过正常窗口输入模拟点击，不读取或修改游戏内存，不注入进程，不隐藏自身，不实现反检测或绕过安全机制。
- 点击区域抖动仅用于覆盖按钮热区、验证不同落点和适配窗口尺寸，不作为规避检测能力。
- 提供前台稳定模式与后台实验模式；后台模式只向指定游戏窗口发送标准鼠标窗口消息，不抢占用户鼠标。

## 目录结构

- `AutoGachaQA.ps1`：工具主体和可视化控制面板。
- `Start-AutoGachaQA.cmd`：面向非程序用户的一键启动入口（使用英文文件名以兼容 SVN CLI）。
- `README.md`：使用步骤、停止方式、限制和维护说明。
- 不保存截图、日志、账号数据、抽卡结果或其他生成物；临时文件用完即清理。

## 交互与安全约定

- 默认只匹配窗口标题 `龙息: 神寂` 和进程名 `Dragonheir`。
- 开始前必须由用户勾选“确认是 Development Build 测试环境”。
- 每次任务必须有明确的十连次数上限；最后一次抽卡后停在结果页。
- `F8` 暂停/继续，`F9` 立即停止；窗口丢失、按钮等待超时或截图检测异常时自动暂停。
- 不向非目标窗口点击，不自动处理充值、购买、登录或弹窗确认；后台固定等待只允许作用于已严格匹配的 Dragonheir 测试窗口。
- 后台模式不支持最小化；优先使用后台画面检测，DirectX 后台捕获不可用时只能采用保守固定等待，并在界面和日志中明确提示。
- 后台模式首次使用必须只跑 1 次十连，确认当前客户端能接收窗口消息后再扩大次数。

## 修改与验证

- 改动后至少运行 PowerShell 语法解析，并核对启动脚本路径。
- 涉及坐标或识别阈值的修改，先在 Development Build 中以 1 次十连验证，再扩大次数。
- SVN 发布只包含本目录运行所需文件，不包含 Unity `.meta`。
